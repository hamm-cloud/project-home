# Project Home - Voice Interface for Hamm

A real-time voice interface for direct conversation with Hamm, built with clean architecture and sub-second latency.

## Architecture

```
Voice → Groq Whisper STT → OpenClaw Session (Hamm) → ElevenLabs TTS → Audio Stream
```

## Features

- **Sub-second latency**: Optimized for real-time conversation
- **Crystal geode UI**: Beautiful, responsive interface
- **WebSocket streaming**: Smooth bidirectional communication
- **Voice synthesis**: ElevenLabs Ivy for natural speech
- **Speech recognition**: Groq Whisper for accurate transcription

## Tech Stack

### Backend
- FastAPI with WebSocket support
- Groq API (Whisper STT)
- ElevenLabs API (Ivy TTS)
- Python 3.12

### Frontend
- React + Vite
- Tailwind CSS
- WebSocket client
- Web Audio API

## Local Development

### Backend Setup

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Create .env file with your API keys
cp .env.example .env

# Run the server
uvicorn main:socket_app --reload --host 0.0.0.0 --port 8000
```

### Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Visit http://localhost:5173 to use the interface.

## Deployment to Railway

### Backend Deployment

1. Create a new Railway project
2. Connect your GitHub repo
3. Set the root directory to `/backend`
4. Add environment variables:
   - `GROQ_API_KEY`
   - `ELEVENLABS_API_KEY`
   - `ELEVENLABS_VOICE_ID`
   - `OPENCLAW_API_URL`
   - `HAMM_SESSION_ID`
5. Deploy!

### Frontend Deployment

1. Create another Railway service
2. Set root directory to `/frontend`
3. Update the WebSocket URL in `App.jsx` to your backend URL
4. Deploy!

## API Endpoints

- `GET /` - Health check
- `WebSocket /ws` - Main voice communication endpoint
- `Socket.IO /socket.io/` - Alternative Socket.IO endpoint

## Voice Flow

1. User clicks microphone button
2. Browser records audio (WebM/Opus)
3. Audio sent to backend via WebSocket
4. Groq Whisper converts speech to text
5. Text sent to OpenClaw Hamm session
6. Response generated by Hamm
7. ElevenLabs Ivy converts text to speech
8. Audio streamed back to browser
9. Web Audio API plays the response

## Performance Targets

- **STT latency**: <500ms (Groq Whisper)
- **LLM response**: <800ms (Haiku via OpenClaw)
- **TTS generation**: <400ms (ElevenLabs streaming)
- **Total round-trip**: <2s target, <1s goal

## Notes

- Uses Ivy voice (ElevenLabs) for softer, more natural tone
- WebSocket for lowest latency bidirectional communication
- Audio streamed in chunks to start playback ASAP
- Crystal geode aesthetic with smooth animations
